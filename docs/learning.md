# 强化学习项目化学习流程需求文档

## 1. 项目背景

本项目以“合成大西瓜”游戏为学习载体，用于辅助学习强化学习相关知识。

当前已有一个可在 Ubuntu 环境下正常运行的 Python 脚本项目，项目内容包含“合成大西瓜”游戏本体、资源文件、项目文档结构以及强化学习相关模块目录。

本项目的核心目的不是按照传统课本或网课视频的方式学习强化学习，而是尝试一种基于真实项目、学习资料和大模型 Agent 辅助的新型学习流程。

## 2. 总体目标

本项目的最终目标是：学习者能够独立设计出一个强化学习算法，并实现一个可以全自动游玩“合成大西瓜”的智能体。

该目标包含两层含义：

1. 学习者需要真正理解强化学习中的核心知识点。
2. 学习者需要能够将强化学习知识应用到“合成大西瓜”项目中，完成自动游玩任务。

## 3. 学习流程需求

在正式开始学习之前，需要先设计一个大致的强化学习学习规划。

该学习规划需要围绕强化学习知识点展开，并按照一定顺序逐个学习相关内容。

学习过程不以单纯阅读教材、观看网课或被动接受知识为主，而是围绕以下方式展开：

1. 基于当前“合成大西瓜”项目进行学习。
2. 结合强化学习相关学习资料进行理解。
3. 使用大模型 Agent 辅助学习、分析、设计和生成相关内容。
4. 尽量将抽象知识点与游戏项目结合。
5. 通过可视化、脚本实验、项目交互和编程练习获得学习反馈。

## 4. 知识点学习需求

对于强化学习中的每个知识点，需要根据其特点决定学习方式。

### 4.1 适合与项目结合的知识点

如果某个强化学习知识点能够较好地与“合成大西瓜”项目结合，并且比较适合通过可视化方式呈现，则应优先结合项目进行学习。

这类知识点的学习目标包括：

1. 将抽象概念映射到游戏中的具体对象或过程。
2. 通过可视化方式增强理解。
3. 通过游戏运行过程观察知识点的实际效果。
4. 通过项目中的数据、状态或行为获得直观反馈。

### 4.2 适合设计脚本或可视化内容的知识点

对于适合可视化展示的知识点，可以通过 Agent 快速设计相关脚本、可视化工具或实验内容，用于辅助学习者快速理解。

这些脚本或可视化内容应服务于知识理解本身，而不是脱离学习目标单独存在。

在必要情况下，可以结合“合成大西瓜”游戏项目中的 API 使用相关数据或功能。

如果当前游戏项目缺少必要接口，可以通过 Agent 辅助补充接口，以便支撑学习过程。

### 4.3 适合设计编程练习的知识点

对于部分已经学习并初步理解的知识点，可以进一步设计编程题目，要求学习者自行完成相关功能。

这类编程题目应当与“合成大西瓜”项目相关，并尽量能够在游戏中看到运行效果。

编程题目的目标包括：

1. 检查学习者是否真正理解该知识点。
2. 让学习者通过自行编码加深理解。
3. 让学习者通过游戏中的可视化结果获得实时正反馈。
4. 逐步提升学习者独立实现强化学习相关模块的能力。

### 4.4 不适合可视化或编程练习的知识点

对于不太适合可视化展示，或者不适合设计编程题目的强化学习知识点，不需要强行项目化或可视化。

这类知识点的学习重点应放在文字描述、概念理解和逻辑解释上。

其目标是帮助学习者理解知识本身，而不是为了形式上与项目结合而额外设计复杂内容。

## 5. Agent 使用需求

本项目中的大模型 Agent 是学习流程的重要组成部分。

Agent 的主要作用是辅助学习者提升学习效率，而不是完全替代学习者完成学习。

Agent 可用于以下方面：

1. 辅助设计强化学习学习规划。
2. 辅助拆分和组织强化学习知识点。
3. 辅助判断某个知识点是否适合与项目结合。
4. 辅助分析当前项目结构和已有代码。
5. 辅助生成学习用脚本、可视化内容或实验内容。
6. 辅助设计与项目相关的编程练习题。
7. 辅助根据学习进度调整后续学习内容。
8. 辅助将学习内容逐步沉淀到项目中。

本项目使用的 Agent 为 ChatGPTProx20，额度较充足，可以支持较高频率的交互和较大规模的项目生成需求。

## 6. 学习反馈需求

该学习流程需要尽量提供及时、明确、正向的学习反馈。

反馈可以来自以下方面：

1. 游戏画面中的可视化结果。
2. 脚本运行输出。
3. 学习者自行完成编程题后的运行效果。
4. Agent 对学习者代码或理解的检查。
5. 项目阶段性功能的完成情况。
6. 自动游玩能力的逐步提升。

学习反馈的目标是帮助学习者确认自己是否真正理解当前知识点，并激励其继续推进后续学习。

## 7. 项目成果需求

本项目的直接成果是一个能够辅助学习强化学习的“合成大西瓜”项目。

项目最终应服务于以下目标：

1. 支撑学习者逐步学习强化学习知识点。
2. 支撑学习者将强化学习知识应用到游戏项目中。
3. 支撑学习者最终独立设计强化学习算法。
4. 支撑学习者最终实现全自动游玩“合成大西瓜”的智能体。

## 8. 方法验证需求

本项目不仅是一个强化学习作业或游戏 AI 项目，也是一种学习方法的创新性尝试。

该学习流程旨在验证：基于项目、学习资料和大模型 Agent 辅助的方式，是否能够显著提升复杂知识的学习效率。

如果该流程在强化学习学习中取得较好效果，后续可以继续尝试将该学习流程迁移到其他知识领域。
