# 项目编码风格

## 教学型详细注释

当前项目同时承担游戏开发和学习材料的作用。源码优先采用“教学型详细注释”风格，让学习者和后续 agent 可以顺着代码理解运行逻辑。

## 注释要求

- 模块顶部需要说明文件职责、边界和主要依赖。
- 关键类、函数需要写清楚用途、输入输出和在游戏流程中的位置。
- 不只在函数声明处写注释；函数体内的关键实现语句也需要解释。
- 对游戏主流程、物理同步、碰撞合成、输入处理、窗口缩放、渲染顺序、状态字段等逻辑必须标注原因。
- 注释应解释“为什么这样做”和“这行代码影响什么”，避免只复述语法。
- 代码行为变化时，要同步更新附近注释，不能留下过期说明。
- 注释以中文为主；第三方库 API、变量名、路径和命令保持原文。

## 适用范围

- `Main.py`
- `src/daxigua/` 下的游戏本体源码
- 后续新增的游戏接口、表现层、核心规则模块

`practice/` 中的学习者练习代码不强制使用同等密度注释；练习代码可以根据学习阶段逐步增加注释。

## 风格示例

推荐：

```python
# resize 事件可能在一次拖动中连续到来。
# 这里先只记录最后一个尺寸，循环结束后统一处理，减少频繁重建背景和边界。
pending_resize = None
```

不推荐：

```python
# 设置变量为 None
pending_resize = None
```
