# 新增 DQNTrainer 参数更新器

## 修改原因

在 GNN-Q 前向模型、`Transition`、`ReplayBuffer` 和 `RolloutCollector` 完成后，需要实现第一版真正更新模型参数的 DQN 训练组件，用于从经验回放池采样并执行 TD loss 反向传播。

## 主要修改

- 新增 `DQNTrainer`。
- 新增 `DQNTrainerConfig` 和 `DQNTrainStats`。
- 使用标准 DQN target：
  - `current_q = online_model(graph)[action_offset]`
  - `target = reward + gamma * max(target_model(next_graph))`
  - terminal/truncated transition 不 bootstrap。
- 默认 loss 使用 `SmoothL1Loss`。
- 初始化时同步 `target_model`，并冻结 target network 参数。
- 每隔 `target_update_interval` 次更新同步一次 target network。
- 默认启用梯度裁剪 `grad_clip_norm=10.0`。
- 第一版逐条图 forward，不做 GraphBatch。
- 第一版不做 Double DQN。

## 涉及文件

- `src/daxigua_rl/training/dqn.py`: 新增 DQN 参数更新器。
- `src/daxigua_rl/training/__init__.py`: 增加 DQN 相关对象的懒加载导出。
- `src/daxigua_rl/README.md`: 补充当前训练侧接口说明。
- `docs/rl/INTERFACE_V0.md`: 记录 DQNTrainer 接口、target 公式和当前约定。
- `docs/project_map/PROJECT_FILE_INDEX.md`: 更新项目文件索引。

## 验证方式

- `conda run -n python-torch python -m py_compile src/daxigua_rl/training/dqn.py src/daxigua_rl/training/__init__.py`
- 使用 `RolloutCollector` 随机采集经验到 `ReplayBuffer`。
- 使用 `GNNQNetwork` 构造 online/target 双网络。
- 执行 `DQNTrainer.train_step()`，确认 loss、梯度、target 同步和参数更新正常。
- `git diff --check`

## 备注

- 本次仍不实现完整训练脚本、epsilon 衰减、模型保存、评估循环或多进程采样。
- 后续可在本更新器基础上增加 Double DQN、GraphBatch、PER 和 GPU batch 训练。
