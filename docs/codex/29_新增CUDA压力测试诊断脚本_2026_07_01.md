# 新增 CUDA 压力测试诊断脚本

## 修改原因

DQN CUDA 训练期间复现了黑屏和 NVIDIA `Xid 16` / `nvidia-modeset` 错误，但资源监控显示当时并非明显内存或显存耗尽。为了判断问题是否来自训练代码本身，需要一个完全脱离游戏和 RL 链路的 CUDA 计算压力测试。

## 主要修改

- 新增 `tools/cuda_stress_test.py`。
- 脚本不 import `daxigua` 或 `daxigua_rl`，只使用 PyTorch 在 CUDA 上执行连续矩阵乘法。
- 支持通过 `--reserve-gpu-memory-mb` 额外预留显存，用于模拟训练时约 3GB 的显存占用。
- 运行期间持续写入：
  - `metrics.csv`: 迭代数、估算 TFLOPS、进程内存、系统内存、GPU 显存、温度、功耗、利用率。
  - `events.jsonl`: CUDA 初始化、显存预留、NVIDIA/Xid 日志检测、温度阈值停止、异常退出等事件。
  - `journal_kernel_poll.log`: 运行期间轮询到的内核日志。
  - `journal_nvidia_poll.log`: 从内核日志中过滤出的 NVIDIA/Xid 相关行。
  - `preflight.txt` / `postflight.txt`: 测试开始和结束时的系统、驱动、显示状态快照。
  - `summary.json`: 测试摘要。
- 支持 `--stop-on-xid`，便于在检测到 Xid 后尽快停止测试，减少系统继续卡死的概率。
- 支持 `--max-temperature-c`，温度超过阈值时主动停止。

## 涉及文件

- `tools/cuda_stress_test.py`: 新增独立 CUDA 压力测试与日志采集工具。
- `docs/project_map/PROJECT_FILE_INDEX.md`: 登记新工具职责和输出位置。

## 验证方式

- `python3 -m py_compile tools/cuda_stress_test.py`
- `python3 tools/cuda_stress_test.py --help`

## 备注

- 正式压力测试可能再次触发黑屏或驱动异常，建议测试前保存工作内容。
- 该工具用于诊断，不参与训练流程，也不改变训练入口参数。
