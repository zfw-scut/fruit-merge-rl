# 新增单局得分和最佳评估记录

## 修改原因

仅看 loss、reward 或评估平均分，不足以直观看到模型在完整游戏中的得分变化。现在新增两类效果指标：

- 训练采集过程中每完成一局游戏的单局分数。
- 定期评估中的最高单局分数和历史最高评估分。

## 主要改动

- 新增 `episode_metrics.csv`，每当 rollout collector 完成一局 episode 就写一行。
- 扩展 `metrics.csv`，增加 `eval_score_max`、`eval_score_min`、`best_eval_score` 和 `best_eval_update`。
- 评估刷新历史最高单局分数时，额外保存 `checkpoints/best.pt`。
- 训练曲线图的 episode score 子图增加训练单局分数、评估最高分和历史最好分。
- 新增 `tests/test_training_metrics.py`，验证 episode 日志和评估最高/最低分。

## 输出文件

```text
runs/xxx/
├── metrics.csv
├── episode_metrics.csv
├── checkpoints/
│   ├── latest.pt
│   ├── best.pt
│   └── step_XXXXXXXX.pt
└── plots/
    └── training_curves.png
```

## 指标含义

- `episode_metrics.csv.score`: 训练采集过程中每局完整游戏的最终分数。
- `eval_score_mean`: 本次评估多局平均分。
- `eval_score_max`: 本次评估多局中的最高单局分数。
- `best_eval_score`: 当前 run 到目前为止出现过的最高评估单局分数。
- `best.pt`: 刷新 `best_eval_score` 时保存的模型 checkpoint。

## 验证

- `EpisodeLogger` 单元测试。
- `evaluate_policy()` 最高/最低分字段测试。
- 小规模训练入口 smoke，确认 `episode_metrics.csv`、`metrics.csv`、`best.pt` 和曲线图输出正常。
