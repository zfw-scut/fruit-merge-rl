# 新增 TOML 训练配置入口

## 修改原因

原 shell 启动脚本把大量训练参数写成环境变量和命令行片段，可读性较差。现在改为 TOML 配置文件承载参数，启动器只保留最薄的一层执行逻辑，让训练配置更接近“选项 = 数值”。

## 主要修改

- 新增 `configs/train_dqn_fast30_parallel.toml`，集中保存正式 fast30 并行训练参数。
- 新增 `scripts/train_dqn.sh`，默认读取上述 TOML 配置并启动训练。
- `train_dqn.py` 新增 `--config` 参数，支持从 TOML 文件读取默认值，并允许命令行参数覆盖配置。
- 删除旧的 `scripts/train_dqn_fast30_parallel.sh`，避免两套启动方式并存。

## 涉及文件

- `configs/train_dqn_fast30_parallel.toml`: 正式训练配置文件。
- `scripts/train_dqn.sh`: 极简训练启动器。
- `src/daxigua_rl/scripts/train_dqn.py`: 新增 TOML 配置读取和字段校验。
- `tests/test_training_metrics.py`: 增加 TOML 配置解析测试。
- `docs/project_map/PROJECT_FILE_INDEX.md`: 同步新增配置和启动入口。
- `docs/rl/TRAINING_SPEED_OPTIMIZATION_PLAN.md`: 更新推荐启动方式。

## 验证方式

- `bash -n scripts/train_dqn.sh`
- `PYTHONPATH=src conda run --no-capture-output -n python-torch python -m unittest discover -s tests`
- 使用极小训练参数通过 `scripts/train_dqn.sh` 做启动烟测。

## 备注

- TOML 中字段名使用 argparse 的内部参数名，也就是命令行参数去掉 `--` 后把短横线改成下划线，例如 `--total-updates` 对应 `total_updates`。
- 如需临时覆盖配置，可在启动命令后继续追加普通训练参数，例如 `./scripts/train_dqn.sh configs/train_dqn_fast30_parallel.toml --total-updates 200000`。
