# 新增 Transition 经验结构

## 修改原因

在 GNN-Q 前向链路跑通后，下一步需要为 replay buffer 和 DQN 更新逻辑准备统一的经验记录结构。`Transition` 用来保存一次环境动作产生的最小训练数据。

## 主要修改

- 新增 `daxigua_rl.training` 包。
- 新增 `Transition` 数据结构，保存：
  - 当前状态图 `graph`
  - 被选择动作在 Q 值张量中的下标 `action_offset`
  - 即时奖励 `reward`
  - 下一状态图 `next_graph`
  - `terminated` 和 `truncated` 标记
- `Transition` 不依赖 PyTorch，保存框架无关 `GraphData`。
- 补充基础一致性检查，避免 action 下标、next graph 等训练数据错位。

## 涉及文件

- `src/daxigua_rl/training/__init__.py`: 新增训练侧数据结构入口。
- `src/daxigua_rl/training/transition.py`: 新增 `Transition`。
- `src/daxigua_rl/__init__.py`: 导出 `Transition`。
- `src/daxigua_rl/README.md`: 补充当前训练侧接口说明。
- `docs/rl/INTERFACE_V0.md`: 记录 `Transition` 字段和 target 计算约定。
- `docs/project_map/PROJECT_FILE_INDEX.md`: 更新项目文件索引。

## 验证方式

- `python -m py_compile src/daxigua_rl/training/__init__.py src/daxigua_rl/training/transition.py src/daxigua_rl/__init__.py`
- 使用 `DaxiguaEnv`、`GraphBuilder` 构造一次真实环境 step，并创建 `Transition`。
- `git diff --check`

## 备注

- 本次不实现 replay buffer、batch 采样、target network、loss 或 optimizer。
- 当前约定 terminal/truncated transition 不进行 bootstrap；后续如果引入时间限制型 truncation，可以在训练逻辑中重新定义。
