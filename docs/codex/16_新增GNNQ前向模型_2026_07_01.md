# 新增 GNN-Q 前向模型

## 修改原因

在 `GraphBuilder`、图特征消融层和节点/边特征精简完成后，需要先跑通“游戏状态图 -> 模型输出动作 Q 值”的前向链路，为后续 DQN 训练循环做准备。

## 主要修改

- 新增 PyTorch 张量转换层：
  - `GraphTensor`
  - `graph_to_tensor(graph)`
- 新增最小 GNN-Q 模型：
  - `GNNQNetwork`
  - `MessagePassingLayer`
- 模型采用统一图 message passing：
  - 节点和边分别通过 MLP 编码。
  - 每层使用 `source node + target node + edge` 生成 message。
  - 按目标节点做 mean aggregation。
  - 使用残差和 `LayerNorm` 更新节点表示。
  - 最后只读取 action 节点，输出 `[action_count]` 个 Q 值。
- PyTorch 相关模块没有放进 `daxigua_rl` 顶层自动导入，避免非训练环境缺少 torch 时影响游戏/环境接口。

## 涉及文件

- `src/daxigua_rl/graph/tensor.py`: 新增 `GraphData -> GraphTensor` 转换。
- `src/daxigua_rl/models/__init__.py`: 新增模型包导出。
- `src/daxigua_rl/models/gnn_q.py`: 新增 GNN-Q 前向模型。
- `docs/project_map/PROJECT_FILE_INDEX.md`: 更新项目文件索引。
- `docs/rl/INTERFACE_V0.md`: 补充模型前向接口说明。
- `docs/rl/gnn_daxigua_design_reference.md`: 补充当前最小 GNN-Q 模型结构。

## 验证方式

- `python -m py_compile src/daxigua_rl/graph/tensor.py src/daxigua_rl/models/__init__.py src/daxigua_rl/models/gnn_q.py`
- `conda run -n python-torch python -m py_compile src/daxigua_rl/graph/tensor.py src/daxigua_rl/models/__init__.py src/daxigua_rl/models/gnn_q.py`
- 使用 `DaxiguaEnv` 生成真实状态，经过 `GraphBuilder`、`graph_to_tensor`、`GNNQNetwork` 得到 `q_values`。
- 验证 `q_values.shape == [15]`，并执行 `q_values.sum().backward()` 确认可反向传播。
- 在非 torch 默认环境中验证 `import daxigua_rl` 不会被模型依赖影响。
- `git diff --check`

## 备注

- 当前 Q 值来自随机初始化模型，没有策略意义。
- 本次不包含 replay buffer、epsilon-greedy、target network、loss 或训练循环。
