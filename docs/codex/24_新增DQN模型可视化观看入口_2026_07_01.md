# 新增 DQN 模型可视化观看入口

## 修改原因

训练入口已经能够保存 DQN checkpoint，但仅靠 `metrics.csv`、评估分数和曲线图还不能直观看出模型在真实游戏画面中的操作质量。项目最终效果需要以原 pygame 游戏窗口为准，因此新增一个“加载模型并观看实际游玩”的入口。

## 主要改动

- 新增 `src/daxigua_rl/playable_adapter.py`，把正在运行的 pygame `Board` 转换成 RL 侧已有的 `GameState` 和 `ActionCandidate` 数据结构。
- 新增 `src/daxigua_rl/scripts/watch_dqn.py`，加载训练 checkpoint，重建 `GNNQNetwork`，并在原游戏窗口上用模型自动选择投放位置。
- 更新 `src/daxigua_rl/README.md`、`docs/project_map/PROJECT_FILE_INDEX.md` 和 `docs/rl/INTERFACE_V0.md`，记录真实窗口观看入口和代码边界。

## 设计约定

- 游戏本体仍然不 import `daxigua_rl`。
- 训练、环境、图构建和模型代码继续走 `HeadlessGame` / `DaxiguaEnv` 路径。
- `watch_dqn.py` 是视觉检查入口，可以在 RL 侧懒加载 `daxigua.app.Board`，但该依赖不进入训练主链路。
- 观看脚本复用原游戏的投放、渲染、碰撞、合成和窗口逻辑，避免另做一套与真实游戏不一致的可视化。

## 使用方式

```bash
PYTHONPATH=src conda run --no-capture-output -n python-torch python -u -m daxigua_rl.scripts.watch_dqn \
  --checkpoint runs/dqn_baseline_h128_l3_10k_eps10k/checkpoints/latest.pt
```

调试时可追加：

```bash
--print-actions
```

用于打印每次模型选择的动作位置和 Q 值摘要。

## 验证

- 通过 `py_compile` 检查新增 Python 文件语法。
- 通过 `watch_dqn --help` 检查命令行入口和懒加载逻辑。
- 通过轻量 fake board 检查 `playable_adapter.py` 能生成 `GameState` 和动作候选。
