# 精简 GNN 节点特征

## 修改原因

在实现 `GraphBuilder` 后，对节点特征进行了第一轮人工审查。目标是让第一版 GNN 输入保持克制，优先保留能直接解释合成、堆叠、死亡风险和动作价值的特征，减少弱相关或重复特征对训练带来的噪声。

## 主要修改

- 真实水果节点删除旋转和年龄相关弱特征：
  - `angle_sin`
  - `angle_cos`
  - `angular_velocity`
  - `age`
- 候选动作节点删除重复或弱区分特征：
  - `y`
  - `drop_x`
  - `normalized_drop_x`
  - `current_level`
  - `current_radius`
- 全局节点删除历史/训练反馈类特征：
  - `score`
  - `last_score`
  - `step_count`
  - `done`
- 待投放队列节点特征暂不调整，继续保留 `level`、`radius`、`queue_index`、`is_current_queue_fruit`。
- 边界节点特征全部保留。

精简后，节点特征维度从早期的 `40` 降到 `28`。当前节点特征更偏向“当前可观察局面”，避免把奖励、历史进程或弱物理量混入模型输入。

## 涉及文件

- `src/daxigua_rl/graph/schema.py`: 精简 `NODE_FEATURE_NAMES`。
- `src/daxigua_rl/graph/builder.py`: 同步调整真实水果节点、动作节点和全局节点的特征生成逻辑，并清理无用归一化配置。

## 验证方式

- `python -m py_compile src/daxigua_rl/graph/schema.py src/daxigua_rl/graph/builder.py`
- 使用 `HeadlessGame` 构造初始局面和投放后一帧稳定局面，调用 `GraphBuilder().build(...)` 验证构图正常。
- 验证结果：节点特征维度为 `28`，边特征维度为 `35`。
- `git diff --check`

## 备注

- `current_level` 和 `current_radius` 不再作为动作节点自身特征输出，但仍会作为 `ActionCandidate` 的原始字段用于计算动作与场上水果之间的边特征。
- 游戏核心状态中的 `angular_velocity` 和 `age_frames` 仍保留，因为它们仍用于稳定判定和原始状态快照；本次只移除 GNN 输入图中的对应节点特征。
