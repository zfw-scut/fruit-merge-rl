# 重构项目包结构

## 修改原因

根目录此前直接放置 `Main.py`、`Game.py`、`Fruit.py`，随着游戏表现层、窗口 resize、文档和后续自动游玩计划增加，根目录职责变得混杂。本次先完成包结构迁移，让游戏本体、资源和未来自动游玩/RL 代码拥有明确边界。

## 主要修改

- 新增 `src/daxigua/` 作为游戏本体包。
- 将原 `Main.py` 的主要实现迁移到 `src/daxigua/app.py`。
- 将原 `Game.py` 迁移到 `src/daxigua/core/board.py`。
- 将原 `Fruit.py` 迁移到 `src/daxigua/core/fruit.py`。
- 根目录 `Main.py` 改为薄入口，保持 `python Main.py` 运行方式不变。
- 新增 `src/daxigua/config.py`，集中放置项目路径、窗口尺寸和 FPS 等基础配置。
- 新增 `src/daxigua_rl/` 作为后续自动游玩/RL 代码预留包，并明确依赖方向只能是 `daxigua_rl -> daxigua`。
- 将水果图片资源从旧 `res/` 迁移到 `assets/fruits/`，运行时不再需要手动解压。
- 将原始资源压缩包移动为 `assets/fruits.zip` 作为归档。
- 新增 `src/daxigua/presentation/` 和 `src/daxigua/utils/` 作为后续拆分表现层和工具层的预留包。

## 涉及文件

- `Main.py`: 改为兼容入口。
- `src/daxigua/app.py`: 游戏应用和当前表现层实现。
- `src/daxigua/core/board.py`: 游戏核心物理和合成逻辑。
- `src/daxigua/core/fruit.py`: 水果定义和资源加载。
- `src/daxigua/config.py`: 基础配置。
- `src/daxigua_rl/README.md`: 后续自动游玩/RL 边界说明。
- `assets/fruits/`: 水果图片资源。
- `assets/fruits.zip`: 原始资源归档。
- `README.md`: 更新运行方式和文件说明。
- `docs/project_map/PROJECT_FILE_INDEX.md`: 更新项目文件索引。

## 验证方式

已执行：

```bash
python -m py_compile Main.py src/daxigua/app.py src/daxigua/core/board.py src/daxigua/core/fruit.py src/daxigua/config.py
SDL_VIDEODRIVER=dummy python -c "from Main import main; from daxigua.app import Board; b=Board(); b.drop_ready_at=0; b._drop_current(); b._resize_window(620, 920); [b.next_frame() for _ in range(5)]; print('smoke ok', b.RES, len(b.segments), len(b.balls), len(b.fruits))"
PYTHONPATH=src python -c "import daxigua; from daxigua.app import Board; print('package import ok', Board.__name__)"
```

结果通过。根目录兼容入口、包内 import、资源路径、投放、resize 和基础帧循环均正常。

## 备注

本次是结构迁移，不继续拆分 `app.py` 内部的渲染、输入、音效和特效逻辑。下一步可以把这些内容逐步移动到 `src/daxigua/presentation/`。
