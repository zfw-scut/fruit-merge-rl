# 优化游戏画面和操作体验

## 修改原因

原手动游戏入口仍偏向调试原型：直接使用 pymunk debug 绘制，鼠标点击后才生成水果，缺少当前水果跟随、预览、下一个水果、合成反馈和正式 UI。此次修改暂时不考虑 RL 训练，只优先提升游戏本体体验。

## 主要修改

- 重写 `Main.py` 的手动游戏表现层，移除 `debug_draw`。
- 新增正式背景、容器、警戒线、HUD、分数和下一个水果预览。
- 当前水果改为跟随鼠标横向移动，增加投放预览线和短冷却。
- 支持鼠标投放，也保留键盘左右移动、空格/回车投放、`R` 重开、`Esc` 退出。
- 新增投放和合成时的粒子、扩散波纹、飘字、轻微屏幕震动和简易音效。
- 调整物理手感，提高重力、降低弹性、提高摩擦和空间迭代数。
- 为水果图片加载增加缓存，减少频繁创建水果时的重复磁盘读取和缩放。
- 在 `GameBoard` 合成逻辑中加入 `on_fruit_merged` 钩子，供表现层响应合成事件。

## 涉及文件

- `Main.py`: 重写手动游戏入口、渲染、交互和反馈效果。
- `Game.py`: 增加合成事件钩子，调整基础碰撞材质参数。
- `Fruit.py`: 增加图片缓存，统一复用缩放后的水果贴图。
- `docs/project_map/PROJECT_FILE_INDEX.md`: 更新文件职责说明。

## 验证方式

已在 `python-torch` conda 环境中执行：

```bash
conda run -n python-torch python -m py_compile Main.py Game.py Fruit.py
SDL_VIDEODRIVER=dummy conda run -n python-torch python -c "from Main import Board; b=Board(); b.drop_ready_at=0; b._drop_current(); [b.next_frame() for _ in range(15)]; print('conda ok', len(b.balls), len(b.fruits), b.score)"
```

结果通过，投放后游戏循环正常。无窗口测试环境会输出 ALSA 音频设备提示，不影响正常桌面运行。

## 备注

本次修改以手动游戏体验为目标，没有维护 RL 训练截图观测的一致性。后续 RL 环境计划重写时，应重新设计训练环境和观测渲染。
