# 创建项目文档结构

## 修改原因

项目后续会继续进行游戏表现、交互、环境兼容和强化学习相关修改，需要一个固定位置记录较大改动，也需要一个文件索引帮助后续 agent 快速预览项目。

## 主要修改

- 新增 `docs/` 作为项目文档根目录。
- 新增 `docs/codex/`，用于记录 Codex 后续较大项目修改。
- 新增 `docs/project_map/`，用于维护项目文件构成、职责分工和可复用组件索引。
- 为两个文档目录分别加入维护规则。
- 生成首版项目文件索引。

## 涉及文件

- `docs/README.md`: 文档目录入口说明。
- `docs/codex/RULES.md`: Codex 修改记录规则。
- `docs/codex/01_创建项目文档结构_2026_06_27.md`: 本次修改记录。
- `docs/project_map/RULES.md`: 项目文件索引维护规则。
- `docs/project_map/PROJECT_FILE_INDEX.md`: 首版项目文件职责索引。

## 验证方式

- 已创建目录和 Markdown 文档。
- 后续通过 `git status --short` 检查新增文件是否符合预期。

## 备注

本次修改只新增文档结构，不改变游戏运行逻辑。
