# 新增 GraphBatch 张量化训练链路

## 修改原因

当前 DQN 训练在每个 `train_step()` 中逐条处理 replay buffer 采样出的图，`batch_size=32` 时会多次调用 online/target GNN。为了给后续 GPU 加速打基础，需要把多张图拼成不连通大图，减少训练侧的小图调度，并让 replay buffer 主链路直接保存张量化经验。

## 主要改动

- 在 `src/daxigua_rl/graph/tensor.py` 新增 `GraphBatch` 和 `collate_graph_tensors()`。
- 在 `src/daxigua_rl/training/tensor_transition.py` 新增 `TensorTransition`，保存 CPU `GraphTensor`。
- 调整 `RolloutCollector`，采集时将 `GraphData` 立即转换为 CPU `GraphTensor`，并写入 `TensorTransition`。
- 调整 `DQNTrainer`，一次 train step 中分别为当前状态和下一状态构建 `GraphBatch`，执行 batched online/target forward。
- 调整 `ReplayBuffer`，让它只负责保存经验对象，不再强制限制为旧 `Transition`。
- 更新模型 `GNNQNetwork.forward()`，支持 `GraphData`、`GraphTensor` 和 `GraphBatch`。
- 新增 `tests/test_graph_batch_training.py`，覆盖 GraphBatch 前向一致性和张量化训练更新。

## 当前训练数据流

```text
DaxiguaEnv
-> GraphBuilder.build(...)
-> graph_to_tensor(...)
-> TensorTransition
-> ReplayBuffer
-> collate_graph_tensors(...)
-> GNNQNetwork(GraphBatch)
-> DQN loss / backward
```

## 设计约定

- `Transition` 保留为框架无关调试结构。
- 正式训练主链路使用 `TensorTransition`。
- replay buffer 长期保存 CPU tensor，不把全部经验放到 GPU。
- GPU 训练时，由模型 forward 将当前 `GraphBatch` 整体移动到模型设备。
- GraphBatch 不改变图结构，只做不连通拼接，不会让不同样本之间传递 message。

## 验证

- `py_compile` 通过。
- `python -m unittest discover -s tests` 通过。
- 极小训练入口 smoke 通过。
- 本地 CPU profiling 显示当前环境 `torch.cuda.is_available() == False`；GraphBatch 路径已可运行，但 CPU 上提速有限，主要价值是后续 GPU batch 训练和减少重复张量转换。
