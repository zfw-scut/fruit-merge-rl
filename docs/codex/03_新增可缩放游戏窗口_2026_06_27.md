# 新增可缩放游戏窗口

## 修改原因

手动游戏窗口原本固定为 `400x800`。为了让玩家可以通过拖动窗口边框改变游戏场地大小，需要让 pygame 窗口支持缩放，并让 pymunk 物理边界跟随窗口尺寸变化。

## 主要修改

- 手动游戏窗口启用 `pygame.RESIZABLE`。
- 新增窗口 resize 事件处理，拖动窗口边框时实时更新游戏宽高。
- `GameBoard` 新增 `resize_world()`，用于重设画布尺寸、投放线高度和物理边界。
- `init_segment()` 改为先移除旧墙体，再创建新的左右墙和地板，避免隐藏物理墙累积。
- resize 事件路径不再调用 `pygame.display.set_mode()`，避免拖拽窗口边框时 SDL 反复重建窗口。
- 同一帧内的多个 resize 事件会被合并，只处理最后一次尺寸变化，降低拖动窗口过程中的卡顿和抖动。
- 背景渐变生成改为 1 像素宽渐变缩放，减少 resize 时 Python 层绘制成本。
- 窗口扩大时保留已有水果位置和速度，让水果在扩展后的新边界中自然继续运动。
- 窗口缩小时对越界水果做轻量夹取，避免水果被新边界卡在场外。
- 背景、顶部面板、游戏池、HUD 和投放预览线会随窗口尺寸重新绘制。

## 涉及文件

- `Main.py`: 启用可缩放窗口，处理 resize 事件，刷新背景和布局。
- `Game.py`: 增加动态重建物理世界边界的基础能力。
- `docs/project_map/PROJECT_FILE_INDEX.md`: 更新文件职责说明。

## 验证方式

已在 `python-torch` conda 环境中执行：

```bash
conda run -n python-torch python -m py_compile Main.py Game.py Fruit.py
SDL_VIDEODRIVER=dummy conda run -n python-torch python -c "from Main import Board; b=Board(); b.drop_ready_at=0; b._drop_current(); b._resize_window(620, 920); [b.next_frame() for _ in range(8)]; b._resize_window(380, 620); [b.next_frame() for _ in range(8)]; print('resize ok', b.RES, len(b.segments), len(b.balls), len(b.fruits))"
SDL_VIDEODRIVER=dummy conda run -n python-torch python -c "import pygame as pg; from Main import Board; b=Board(); original=pg.display.set_mode; pg.display.set_mode=lambda *a, **k: (_ for _ in ()).throw(RuntimeError('set_mode called')); b._resize_window(620, 920); [b.next_frame() for _ in range(3)]; print('event resize path ok', b.RES, len(b.segments)); pg.display.set_mode=original"
SDL_VIDEODRIVER=dummy conda run -n python-torch python -c "import pygame as pg; from Main import Board; b=Board(); calls=[]; original=b._resize_window; b._resize_window=lambda w,h,recreate_display=False: (calls.append((w,h)), original(w,h,recreate_display)); [pg.event.post(pg.event.Event(pg.VIDEORESIZE, {'size': (400+i*5, 700+i*3), 'w': 400+i*5, 'h': 700+i*3})) for i in range(20)]; b._handle_events(); print('resize calls', len(calls), 'last', calls[-1], 'res', b.RES)"
```

结果通过，窗口缩放后物理边界数量保持为 3，已有水果和游戏循环正常；普通 resize 路径不会调用 `set_mode()`。

## 备注

当前最小窗口尺寸限制为 `360x560`，避免 UI 和投放范围过度挤压。
