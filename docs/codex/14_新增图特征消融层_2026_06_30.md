# 新增图特征消融层

## 修改原因

后续需要对简单模型和高级模型进行对比，并可能临时屏蔽部分图特征做消融研究。为了避免反复修改 `GraphBuilder` 或 `schema.py`，新增独立消融层，在完整图构建完成后按配置把部分特征置零。

## 主要修改

- 新增 `GraphAblator`，用于对 `GraphData` 应用特征遮罩。
- 新增 `FeatureAblationConfig` 和 `FeatureMask`，支持：
  - 对所有节点/边禁用指定特征。
  - 只对指定节点类型或边类型禁用指定特征。
  - 通过预定义特征组快速配置常见实验。
- 新增消融预设：
  - `full`
  - `no_board_motion`
  - `no_global_summary`
  - `no_queue_planning`
  - `no_action_board_relation`
- 消融只会置零特征值，不改变节点数量、边数量、节点编号、边编号或特征维度。

## 涉及文件

- `src/daxigua_rl/graph/ablation.py`: 新增图特征消融工具。
- `src/daxigua_rl/graph/__init__.py`: 导出消融相关类型、预设和入口函数。
- `src/daxigua_rl/__init__.py`: 在 RL 包顶层导出常用消融入口，方便训练代码引用。

## 验证方式

- `python -m py_compile src/daxigua_rl/__init__.py src/daxigua_rl/graph/__init__.py src/daxigua_rl/graph/ablation.py src/daxigua_rl/graph/builder.py src/daxigua_rl/graph/schema.py`
- 使用 `GraphAblator.from_preset('no_board_motion')` 验证场上水果的 `vx`、`vy`、`stable` 等特征被置零。
- 使用 `FeatureMask(('action_index',), target_type='action')` 验证只遮罩动作节点，不影响队列节点中的同名字段。
- 验证 `GraphAblator.apply(...)` 不会原地修改原始 `GraphData`。
- `git diff --check`

## 备注

- 边界节点特征没有放入默认消融预设，默认保持完整。
- 后续训练脚本可通过 preset 名称选择消融方案，不需要修改构图逻辑。
