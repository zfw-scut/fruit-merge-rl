# 新增训练资源旁路监控脚本

## 修改原因

正式 DQN 训练时出现过系统内存和 swap 被打满、图形显示异常以及重启前 NVIDIA modeset warning。为了在保持原训练强度的前提下定位问题，需要一个不侵入训练代码的外部监控程序，持续记录训练期间的系统资源变化。

## 主要修改

- 新增独立脚本 `tools/monitor_training_resources.py`。
- 监控器不 import 游戏或 RL 代码，只通过 Linux `/proc` 和 `nvidia-smi` 读取状态，避免影响训练主链路。
- 默认每 3 秒采样一次，并输出：
  - `system_metrics.csv`: 系统内存、swap、目标进程汇总、GPU 汇总。
  - `process_metrics.csv`: 匹配到的训练进程及其子进程 RSS、VMS、CPU、线程数。
  - `gpu_metrics.csv`: 每块 NVIDIA GPU 的显存、利用率、温度、功耗。
  - `gpu_process_metrics.csv`: 当前 CUDA/compute 进程。
  - `events.jsonl`: 低内存、swap 升高、目标进程 RSS 过高、GPU 查询失败等结构化事件。
  - `metadata.json` 和 `summary.json`: 本次监控的启动参数和收尾摘要。
- 监控器默认匹配 `daxigua_rl.scripts.train_dqn`，也支持通过 `--pid` 或重复传入 `--match` 指定目标进程。
- 监控器会排除自身和父进程链，避免启动命令中的 `--match` 参数误匹配到 shell。

## 涉及文件

- `tools/monitor_training_resources.py`: 新增训练旁路资源监控入口。
- `docs/project_map/PROJECT_FILE_INDEX.md`: 登记新工具的职责和输出位置。

## 验证方式

- `python3 -m py_compile tools/monitor_training_resources.py`
- `python3 tools/monitor_training_resources.py --no-gpu --max-samples 2 --interval 0.1 --log-dir /tmp/daxigua_resource_monitor_smoke --match definitely_not_running_train_dqn`
- 在非沙盒环境运行一次 GPU smoke test，确认能通过 `nvidia-smi` 采到 RTX 4070 Laptop GPU 的显存、利用率和温度信息。

## 备注

- 该脚本不会主动限制训练进程内存，也不会在阈值触发时杀进程；第一版只负责记录现场。
- 长时间正式训练时，建议先启动监控器，再启动训练命令，确保训练启动前后的资源变化都被记录。
