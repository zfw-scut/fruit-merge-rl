# 新增 DQN 训练入口和可视化

## 修改原因

在 `RolloutCollector`、`ReplayBuffer`、`DQNTrainer` 和 reward shaping 完成后，需要一个正式训练入口，把采集、训练、epsilon 衰减、日志、保存、评估和可视化串成可长期运行的训练流程。

## 主要修改

- 新增 `src/daxigua_rl/scripts/train_dqn.py`。
- 新增 `src/daxigua_rl/scripts/__init__.py`。
- 训练入口包含：
  - warmup 随机采样。
  - epsilon-greedy 线性衰减。
  - `RolloutCollector.collect_steps()`。
  - `DQNTrainer.train_step()`。
  - 终端日志。
  - `metrics.csv` 指标记录。
  - `latest.pt` 和周期 step checkpoint。
  - 独立 greedy 评估。
  - matplotlib 生成 `training_curves.png`。
- 训练输出默认写入 `runs/dqn_YYYYMMDD_HHMMSS/`。
- `.gitignore` 新增 `runs/`，避免训练产物进入仓库。
- matplotlib 使用 `Agg` 后端，并把 `MPLCONFIGDIR` 指向当前 run 目录，避免用户目录不可写导致缓存警告。

## 涉及文件

- `.gitignore`: 忽略 `runs/`。
- `src/daxigua_rl/scripts/__init__.py`: 新增脚本包。
- `src/daxigua_rl/scripts/train_dqn.py`: 新增 DQN 训练入口。
- `src/daxigua_rl/README.md`: 补充训练命令说明。
- `docs/rl/INTERFACE_V0.md`: 记录训练入口、输出结构和可视化指标。
- `docs/project_map/PROJECT_FILE_INDEX.md`: 更新项目文件索引。

## 验证方式

- `conda run -n python-torch python -c "import matplotlib; print(matplotlib.__version__)"`
- `conda run -n python-torch python -m py_compile src/daxigua_rl/scripts/__init__.py src/daxigua_rl/scripts/train_dqn.py`
- 运行短训练：

```bash
PYTHONPATH=src conda run -n python-torch python -m daxigua_rl.scripts.train_dqn \
  --total-updates 3 \
  --warmup-steps 8 \
  --collect-per-update 1 \
  --batch-size 4 \
  --replay-capacity 64 \
  --log-interval 1 \
  --save-interval 2 \
  --eval-interval 2 \
  --eval-episodes 1 \
  --eval-max-steps 5 \
  --plot-interval 1 \
  --hidden-dim 16 \
  --message-layers 1 \
  --run-dir /tmp/daxigua_train_smoke_codex
```

- 验证生成：
  - `metrics.csv`
  - `config.json`
  - `checkpoints/latest.pt`
  - `checkpoints/step_00000002.pt`
  - `plots/training_curves.png`
- `git diff --check`

## 备注

- 第一版训练入口仍为单进程同步训练。
- 当前不包含 TensorBoard、多进程采样、GraphBatch、Double DQN 或断点恢复。
- `metrics.csv` 是主要可视化数据源，png 可随时从指标重新生成。
