# 新增无渲染 RL 接口

## 修改原因

后续将开始强化学习模型搭建，需要先提供稳定、无渲染、可训练的游戏接口。同时需要保持游戏本体和 RL 代码职责分离，避免训练逻辑直接依赖 pygame 手动游戏入口。

## 主要修改

- 新增 `src/daxigua/core/rules.py`，集中管理水果等级、半径、队列长度、生成范围、合成计分和物理半径规则。
- 新增 `src/daxigua/core/state.py`，定义 `GameState`、`FruitState`、`ActionCandidate`、`DropResult`、`PhysicsResult` 等纯数据结构。
- 新增 `src/daxigua/core/engine.py`，提供 `HeadlessGame` 无渲染游戏引擎。
- 新增 `src/daxigua_rl/env.py`，提供隔离的 `DaxiguaEnv` 环境壳层。
- 手动游戏水果半径和队列长度改为复用 `core.rules`，减少规则重复。
- 新增 `docs/rl/INTERFACE_V0.md`，记录 v0 接口边界和使用方式。

## 验证方式

- 执行 Python 编译检查，覆盖游戏本体和 RL 包新增文件。
- 执行无渲染 `HeadlessGame` smoke test，确认 reset、动作候选、投放、物理推进、状态返回可用。
- 执行 `DaxiguaEnv` smoke test，确认 reset/step 返回值结构正常。
- 执行手动游戏 dummy smoke test，确认 pygame 入口仍可创建并投放。

## 备注

本次不加入模型、GNN 图构建器、replay buffer 或训练循环。验证使用命令行临时片段完成，没有向仓库新增临时测试脚本。
