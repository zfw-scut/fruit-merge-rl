# 优化 RL 奖励计算

## 修改原因

原先 `DaxiguaEnv.step()` 的 reward 只由 `score_delta` 和死亡惩罚组成，大量没有合成的投放会得到 `0`，奖励信号较稀疏，不利于早期 DQN 训练。

## 主要修改

- 新增独立 reward 模块，不把训练奖励逻辑写入游戏本体。
- 新增 `RewardConfig`，统一配置奖励权重。
- 新增 `RewardBreakdown`，用于记录每一步奖励明细。
- `DaxiguaEnv.step()` 改为根据动作前后状态计算 reward shaping。
- 新 reward 包含：
  - 合成分数奖励。
  - 存活小奖励。
  - 高度变化奖励/惩罚。
  - 危险高度持续惩罚。
  - 游戏失败终局惩罚。
- `info["reward_breakdown"]` 会返回本次 reward 的各项组成，方便训练日志分析。

## 涉及文件

- `src/daxigua_rl/reward.py`: 新增 reward shaping 逻辑。
- `src/daxigua_rl/env.py`: 接入 `compute_reward()`。
- `src/daxigua_rl/__init__.py`: 导出 reward 相关接口。
- `src/daxigua_rl/README.md`: 补充 reward 配置说明。
- `docs/rl/INTERFACE_V0.md`: 更新 reward 公式和默认配置。
- `docs/project_map/PROJECT_FILE_INDEX.md`: 更新项目文件索引。

## 验证方式

- `conda run -n python-torch python -m py_compile src/daxigua_rl/reward.py src/daxigua_rl/env.py src/daxigua_rl/__init__.py`
- 使用 `DaxiguaEnv` 执行真实 step，确认返回 reward 和 `reward_breakdown`。
- `git diff --check`

## 备注

- 当前 reward shaping 是第一版默认值，后续需要结合训练曲线继续调参。
- 暂未加入同级靠近奖励、连锁合成潜力奖励或待投放队列诱导奖励，避免第一版人为策略引导过重。
