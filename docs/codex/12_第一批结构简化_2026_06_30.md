# 第一批结构简化

## 修改原因

项目进入 RL 接口搭建后，部分早期小游戏结构和预留目录已经成为阅读负担。为了在继续做更大的核心架构重构前先降低复杂度，本次只处理低风险的历史包袱和空占位。

## 主要修改

- 简化 `src/daxigua/core/fruit.py`，删除 11 个重复水果子类，改为单一 `Fruit` 显示精灵。
- 保留 `create_fruit(level, x, y)` 工厂入口，减少外部调用改动。
- `app.py` 不再通过创建临时水果对象获取半径，改为直接读取 `rules.py`。
- 删除固定窗口后不再使用的 `app.py` resize 辅助函数。
- 删除 `board.py` 中旧调试绘制字段、旧 `show_score()` 和空 `run()`。
- 删除空的 `src/daxigua/presentation/` 和 `src/daxigua/utils/` 预留包。
- 更新项目文件索引，移除已经删除的空预留包说明。

## 涉及文件

- `src/daxigua/core/fruit.py`
- `src/daxigua/app.py`
- `src/daxigua/core/board.py`
- `src/daxigua/presentation/__init__.py`
- `src/daxigua/utils/__init__.py`
- `docs/project_map/PROJECT_FILE_INDEX.md`

## 验证方式

- 执行 Python 编译检查，覆盖当前所有源码入口。
- 执行 HeadlessGame smoke test，确认 reset、动作候选、投放、物理推进和状态返回正常。
- 执行 DaxiguaEnv smoke test，确认 reset/step 返回结构正常。
- 执行手动游戏 dummy smoke test，确认 pygame 入口仍可创建、投放并推进帧循环。
- 执行 `git diff --check`，确认没有空白格式问题。

## 备注

本次不处理 `GameBoard` 和 `HeadlessGame` 的双物理逻辑。该部分属于第二批中风险架构重构，应单独规划和验证。
