# 补充源码教学注释

## 修改原因

项目后续会作为强化学习项目化学习载体使用，学习者需要先理解游戏本体的文件结构、运行流程和核心规则。为降低阅读门槛，本次为游戏源码补充详细中文注释，并记录后续源码应遵循的注释风格。

## 主要修改

- 为 `Main.py` 和 `src/daxigua/` 下游戏本体源码补充模块说明、类说明、函数说明和关键语句注释。
- 重点解释了水果创建、物理刚体、碰撞合成、计分、失败检测、输入处理、窗口缩放、渲染顺序、粒子和音效等逻辑。
- 新增 `docs/CODING_STYLE.md`，记录“教学型详细注释”的项目编码风格。
- 更新文档入口和项目文件索引，标注编码风格文档的位置。

## 验证方式

- 执行 Python 编译检查，确认注释补充和少量格式整理没有引入语法错误。
- 使用 dummy 视频驱动创建游戏对象、投放一次水果、调整窗口尺寸并推进数帧，确认主要运行路径仍可执行。

## 备注

本次不改变游戏行为，只增强源码可读性和学习可解释性。
