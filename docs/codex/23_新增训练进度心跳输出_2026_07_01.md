# 新增训练进度心跳输出

## 修改原因

默认训练日志由 `log_interval` 控制，间隔可能较长；训练开始后的 warmup 阶段也可能持续一段时间没有反馈。为了方便观察训练是否仍在推进，需要一个按时间间隔输出的轻量进度提示。

## 主要修改

- `train_dqn.py` 新增 `--progress-interval` 参数，默认 `3.0` 秒。
- warmup 阶段改为分块采集，方便按时间输出进度。
- 训练阶段每隔固定时间打印一次 `[progress]` 心跳。
- 心跳包含：
  - 当前阶段。
  - 当前进度和百分比。
  - 环境步数。
  - replay buffer 大小。
  - epsilon。
  - env steps/s。
  - 最新 loss。
- `--progress-interval 0` 可关闭该功能。
- 通过 conda 运行时需要使用 `conda run --no-capture-output`，否则 conda 会捕获输出，导致进度信息无法实时显示。

## 涉及文件

- `src/daxigua_rl/scripts/train_dqn.py`: 新增进度心跳输出。
- `docs/rl/INTERFACE_V0.md`: 补充训练进度输出说明。

## 验证方式

- `conda run -n python-torch python -m py_compile src/daxigua_rl/scripts/train_dqn.py`
- 使用短训练和 `--progress-interval 0.1` 验证 warmup 和 train 阶段会按时间输出进度。
- `git diff --check`

## 备注

- 进度心跳只打印到终端，不写入 `metrics.csv`，不影响训练曲线数据。
- 详细训练指标仍由 `log_interval` 控制。
