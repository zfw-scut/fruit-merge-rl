# 强化学习项目化学习路线

本路线是第一版学习规划，用于指导后续围绕“合成大西瓜”项目学习强化学习。

路线会随着学习进度调整。当前阶段不急于设计正式游戏接口，也不急于编写训练算法，先把学习问题拆清楚。

## 阶段 1：项目和游戏机制理解

目标：

- 理解当前游戏项目结构。
- 理解游戏循环、投放、碰撞、合成、计分、失败条件。
- 建立“这个游戏中哪些东西可能成为 RL 问题元素”的直觉。

项目结合方式：

- 阅读 `src/daxigua/` 的游戏本体代码。
- 运行游戏，观察水果下落、合成和失败过程。
- 记录玩家决策点：什么时候能操作，能做什么操作。

适合练习：

- 画出游戏运行流程图。
- 用文字描述一次水果投放到合成的过程。
- 列出游戏中可观察的信息和不可观察的信息。

## 阶段 2：强化学习问题定义

目标：

- 理解 agent、environment、state、action、reward、done 的基本含义。
- 能把这些概念初步映射到“合成大西瓜”。
- 区分游戏本体和 RL 环境包装。

项目结合方式：

- 暂不编写正式接口。
- 先用文档描述：如果把游戏看成环境，智能体在每一步能看到什么、能做什么、得到什么反馈。

适合练习：

- 写出 2 到 3 套不同的状态设计草案。
- 写出 2 到 3 套不同的动作空间设计草案。
- 写出奖励函数的初版想法，并说明可能的问题。

## 阶段 3：状态、动作、奖励、终止条件设计

目标：

- 理解一个 RL 问题的建模选择会直接影响学习难度。
- 能比较不同状态表示、动作空间和奖励函数的优缺点。
- 初步意识到“人类觉得简单”和“机器能学会”不是一回事。

项目结合方式：

- 基于游戏规则和画面观察设计多个候选方案。
- 暂时以分析和小脚本实验为主，不进入正式训练。

适合练习：

- 设计离散动作空间，例如固定若干投放点。
- 设计结构化状态，例如水果位置、半径、类型列表。
- 设计图像状态，例如游戏画面截图。
- 比较稀疏奖励和密集奖励。

## 阶段 4：随机策略和基线实验

目标：

- 理解 baseline 的意义。
- 知道在训练算法前，需要先知道随机策略或简单规则策略能达到什么水平。
- 学会用数据而不是感觉评估策略。

项目结合方式：

- 在明确需要的最小接口后，设计随机投放或规则投放实验。
- 记录分数分布、平均分、最高分、游戏时长等指标。

适合练习：

- 写一个随机动作选择器。
- 设计一个简单规则策略，例如优先投到同类水果附近。
- 记录多局结果并画简单统计图。

## 阶段 5：MDP、回报和价值函数

目标：

- 理解 MDP、return、discount factor、value function、Q function。
- 理解为什么“当前选择”会影响未来局面。
- 能将价值函数解释为某个状态或动作的长期预期收益。

项目结合方式：

- 使用简化版游戏状态或玩具环境做可视化。
- 可以设计小型脚本，不必直接接入完整游戏。

适合练习：

- 在一个简化格子环境中计算回报。
- 手算几个状态动作的长期价值。
- 解释为什么即时合成高分不一定代表长期最优。

## 阶段 6：表格型方法

目标：

- 理解 Monte Carlo、TD、SARSA、Q-learning 的基本思想。
- 理解探索和利用的矛盾。
- 理解为什么完整“合成大西瓜”不适合直接使用纯表格方法。

项目结合方式：

- 先在小型离散环境中练习。
- 再讨论如果把游戏粗粒度离散化，会遇到哪些问题。

适合练习：

- 实现一个小型 Q-learning 示例。
- 调整 epsilon，观察探索程度变化。
- 记录学习曲线。

## 阶段 7：函数近似和神经网络

目标：

- 理解为什么状态空间大时需要函数近似。
- 理解神经网络如何近似 Q 函数或策略。
- 理解输入特征设计对学习效果的影响。

项目结合方式：

- 对比结构化状态和图像状态。
- 讨论不同输入会带来的训练复杂度。

适合练习：

- 用简单神经网络拟合一个玩具价值函数。
- 比较手工特征和原始图像输入。

## 阶段 8：DQN 和经验回放

目标：

- 理解 DQN 的基本结构。
- 理解经验回放、目标网络、epsilon-greedy。
- 理解训练不稳定的常见来源。

项目结合方式：

- 在明确接口后，可以开始设计 `src/daxigua_rl/` 中的环境包装。
- 先跑小规模实验，再逐步接近完整游戏。

适合练习：

- 实现 replay buffer。
- 实现 epsilon 衰减。
- 在玩具环境中跑通 DQN。

## 阶段 9：自动游玩智能体

目标：

- 设计并实现可以自动游玩“合成大西瓜”的智能体。
- 建立训练、评估、保存、复现实验的基本流程。
- 能解释智能体行为和训练结果。

项目结合方式：

- 在 `src/daxigua_rl/` 中实现正式环境、策略、训练脚本和评估脚本。
- 与游戏本体通过稳定接口交互。

适合练习：

- 训练一个简单 agent。
- 对比随机策略、规则策略和学习策略。
- 可视化策略表现和分数曲线。

## 阶段 10：学习方法复盘

目标：

- 复盘项目化学习对理解强化学习是否有效。
- 总结哪些知识点适合可视化，哪些适合练习，哪些适合文字解释。
- 为后续迁移到其他学习领域积累方法。

项目结合方式：

- 整理阶段总结。
- 记录 Agent 协作方式中有效和无效的部分。
- 总结项目最终成果和不足。

适合练习：

- 写一份完整项目复盘。
- 总结自己的 RL 知识地图。
- 提出下一轮改进计划。
