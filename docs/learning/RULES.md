# Learning Docs Rules

本文件定义 Agent 生成学习规划、知识点说明、练习任务和阶段总结时应遵守的规则。

## 基本原则

- 学习目标优先于代码产出。
- 先解释为什么学，再解释学什么，最后再决定是否写代码。
- 不强行把所有知识点项目化；适合结合游戏的才结合。
- 不在 `docs/learning/` 中放大段实验代码，代码应进入 `practice/`。
- 不在学习阶段过早设计正式 RL 接口，接口应等学习需求清晰后再抽象。

## 文档生成要求

Agent 生成学习文档时，应尽量包含：

- 当前知识点的学习目标。
- 与“合成大西瓜”项目的关系。
- 是否适合可视化或编程练习。
- 推荐练习方式。
- 学习者完成后的检查标准。
- 可能的误区。

## 练习设计要求

练习题应满足：

- 和当前学习阶段相关。
- 范围可控，可以在较短时间内完成。
- 有明确输入、输出或可观察结果。
- 优先使用 `practice/` 目录，不直接修改正式源码。
- 如果必须使用游戏代码，应通过稳定、清晰的调用方式完成。

## Agent 协作方式

Agent 可以帮助：

- 拆分学习路线。
- 解释知识点。
- 判断知识点是否适合项目化。
- 设计练习题和验收标准。
- 检查学习者练习代码。
- 总结阶段复盘。
- 将成熟代码从 `practice/` 迁移到正式模块。

Agent 不应该：

- 直接替学习者完成所有练习。
- 在没有学习需求支撑时提前堆复杂框架。
- 为了形式感强行可视化所有知识点。
- 让 `src/daxigua/` 反向依赖 `src/daxigua_rl/` 或 `practice/`。

## 文件命名建议

阶段文档：

```text
01_项目和游戏机制理解.md
02_强化学习问题定义.md
03_状态动作奖励设计.md
```

练习说明：

```text
01_观察游戏状态.md
02_随机策略实验.md
03_奖励函数草案.md
```

笔记：

```text
MDP_概念笔记.md
价值函数_常见误区.md
阶段1_复盘.md
```
